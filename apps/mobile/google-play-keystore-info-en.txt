# ==========================================
# HOW TO BUILD THE APP FOR THE PLAY STORE
# ==========================================
Since you are using Expo, the best way to generate the ".aab" (Android App Bundle) file required by Google Play is using EAS Build.

1. Install EAS CLI (if you haven't already):
   npm install -g eas-cli

2. Log in to your Expo account:
   eas login

3. Initialize the project configuration in EAS:
   eas build:configure

4. Generate the final build for the store:
   eas build --platform android --profile production

TIP: When EAS asks if you want to automatically generate a new signing key, you can say "Yes" (recommended, as Expo manages it for you with maximum security) OR you can choose to upload the "release.keystore" file generated here, providing the Alias and Passwords listed above.

# ==========================================
# PUBLISHING ON THE GOOGLE PLAY CONSOLE
# ==========================================
1. Create a developer account on the Google Play Console (costs $25 once).
2. Click on "Create App", fill in the name ("Fiado") and details.
3. Go to "Testing" or "Production", and create a new release.
4. Upload the ".aab" file generated by EAS Build.
5. Fill in the content rating, images, icon, and description.
6. Submit for Google's review!
